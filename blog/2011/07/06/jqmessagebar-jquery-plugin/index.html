jqMessageBar, a jQuery Notification Bar

Jul 6th, 2011

I recently created my first jQuery plugin dubbed jqMessageBar. I was working on a Ruby on Rails application at work and had almost finished re-writing the client side using JavaScriptMVC when I ran into my final “TODO”.

Throughout the client code I had been lazy, or perhaps pragmatic (to be optimistic), and put “TODO: show nice notification” in many of the jQuery.ajax error callbacks (and some of the success callbacks as well). While writing the client I wasn’t sure how I wanted to present “Tag successfully added” and “Failed to add tag…”. But as I dug around stackoverflow I noticed they make frequent use of a notification bar located across the top of the page (I’ve also seen it on the codeschool site as well).

I took a look at the source of those sites and their both just using <div>’s with slightly different CSS applied. The bells went off and I thought, this would be a nice little plugin to write.

jqMessageBar was inspired by those sites but also by jBar. Before I began writing my own plugin I did some searching to see if what I wanted already existed as a convenient plugin. jBar got me started but I needed more control so I set out to write my first jQuery plugin.

The primary features include:

  • display one or more messages with buttons to navigate between them
  • position the message bar at the top or bottom of the page
  • style messages based on a “type” (configurable)
  • easy to use API to interact with the message bar (show, hide, show next, etc)

The source is available on GitHub and the demo page is available here for you to give it a whirl.

Please add bugs/feature requests and/or fork and submit pull requests.

Happy jqMessageBar’ing, or something!
